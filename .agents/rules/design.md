<RULE[meridian_design_code]> SYSTEM PROMPT: MERIDIAN MAIL AESTHETICS & UI
ARCHITECTURE

When generating UI components, styles, or user flows, you MUST strictly follow
the "Aesthetic Precision" philosophy, heavily inspired by Dieter Rams
(Braun/Apple):

1. LESS, BUT BETTER (Воздух и Чистота): Remove all visual noise, unnecessary
   borders, decorative drop shadows, and dividers. Use whitespace ("Fresh Air")
   to separate elements, not lines. The interface should be almost invisible.
2. UNOBTRUSIVE UTILITY (Утилитарность): The app is a tool, not a social network.
   Use a highly neutral color palette (deep blacks, bright whites, clean grays).
   Reserve 1-2 accent colors (e.g., pure blue or orange) ONLY for primary
   actions and "unread" states.
3. ABSOLUTE CLARITY (Диалог, а не Инбокс): Complex backend logic (IMAP/SMTP
   headers, routing) must NEVER bleed into the UI. Strip away formal email
   artifacts (To, Cc, Subject, signatures, quotes) on the main screens. It must
   look and feel like a modern, simple messenger.
4. TIMELESSNESS (Вневременность): Ignore fleeting UI fads (e.g., neomorphism,
   heavy glassmorphism). Rely entirely on strict monochrome typography (Inter /
   San Francisco), high contrast, and mathematically perfect padding/margins.
5. RADICAL HONESTY (Честность): Do not design "fake" indicators. If the
   underlying protocol (IMAP) doesn't guarantee a read receipt, do not draw read
   ticks. Network states, sync progress, and encryption must be transparent but
   quiet. </RULE[meridian_design_code]>
